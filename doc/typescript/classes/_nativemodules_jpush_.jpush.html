Static setLatestNotificationNumber

  • setLatestNotificationNumber(params: object): void
    • Defined in nativeModules/JPush.ts:475

    设置保留最近通知条数 通过极光推送，推送了很多通知到客户端时，如果用户不去处理，就会有很多保留在那里。 调用此 API 可以限制保留的通知条数，默认为保留最近 5 条通知。

    仅对通知有效。所谓保留最近的，意思是，如果有新的通知到达，之前列表里最老的那条会被移除。 例如，设置为保留最近 5 条通知。假设已经有 5 条显示在通知栏，当第 6 条到达时，第 1 条将会被移除。

    Parameters

    • params: object
      • notificationMaxNumber: number

        最多显示的条数

    Returns void

Static setPushTime

  • setPushTime(params: object): void
    • Defined in nativeModules/JPush.ts:444

    设置允许推送时间 默认情况下用户在任何时间都允许推送。即任何时候有推送下来，客户端都会收到，并展示。 开发者可以调用此 API 来设置允许推送的时间。 如果不在该时间段内收到消息，SDK 的处理是：推送到的通知会被扔掉。

    这是一个纯粹客户端的实现，所以与客户端时间是否准确、时区等这些，都没有关系。 而且该接口仅对通知有效，自定义消息不受影响。

    Parameters

    • params: object
      • Optional pushTimeDays?: number[]

        0 表示星期天，1 表示星期一，以此类推。 （ 7 天制，Set 集合里面的 int 范围为 0 到 6 ） pushTimeDays值为 null，则任何时间都可以收到通知 || pushTimeDays值为 []，则表示任何时间都收不到通知

      • Optional pushTimeEndHour?: undefined | number

        允许推送的结束时间 （ 24 小时制：endHour 的范围为 0 到 23 ）

      • Optional pushTimeStartHour?: undefined | number

        允许推送的开始时间 （ 24 小时制：startHour 的范围为 0 到 23 ）

    Returns void

Static setSilenceTime

  • setSilenceTime(params: object): void
    • Defined in nativeModules/JPush.ts:460

    设置通知静默时间 默认情况下用户在收到推送通知时，客户端可能会有震动，响铃等提示。 但用户在睡觉、开会等时间点希望为“免打扰”模式，也是静音时段的概念。 开发者可以调用此 API 来设置静音时段。如果在该时间段内收到消息，则：不会有铃声和震动。

    Parameters

    • params: object
      • silenceTimeEndHour: number

        静音时段的结束时间 - 小时 （ 24 小时制，范围：0~23 ）

      • silenceTimeEndMinute: number

        静音时段的结束时间 - 分钟（范围：0~59 ）

      • silenceTimeStartHour: number

        静音时段的开始时间 - 小时 （ 24 小时制，范围：0~23 ）

      • silenceTimeStartMinute: number

        静音时段的开始时间 - 分钟（范围：0~59 ）

    Returns void
